Les machines de test sont accessibles via SSH par réservation de slots d'une heure. Rendez-vous sur https://studreservation.info.ucl.ac.be/ et connectez-vous avec le login et mot de passe que vous avez reçu par e-mail.

Rendez-vous ensuite sur l'onglet INGI-Student LEPL1503. Vous pourrez ensuite réserver un slot d'une heure sur l'un des trois serveurs disponibles, soit studsrv2.info.ucl.ac.be, studsrv3.info.ucl.ac.be ou studsrv4.info.ucl.ac.be. Chacun de ces serveurs dispose de 8 coeurs, 8GB de ram et 8GB d'espace disque. Vous pourrez réserver un slot au jour et l'heure qui vous convient si personne ne l'a actuellement déjà réservé.  Vous aurez ensuite accès aux machines durant le slot choisi (Il se peut que vous deviez attendre jusqu'à deux minutes après le début du slot avant d'avoir effectivement accès à la machine).

La machine est accessible via SSH, et vous devrez utiliser la passerelle studssh.info.ucl.ac.be pour pouvoir atteindre votre serveur de test. Nous vous proposons deux manières simples pour vous connecter à ces serveurs, à vous de choisir celle qui vous convient le mieux.


Première méthode: se connecter en utilisant un script fourni
============================================================

Nous vous fournissons un script shell pour pouvoir vous connecter simplement à la machine en ligne de commande. Il suffit d'exécuter le script comme suit :

	sh connect_to_experiments_server.sh {USERNAME}@{SERVERNAME} {PATH_TO_SSH_KEY}

{USERNAME} est le nom d'utilisateur (de la forme guestXXX) que vous utilisez pour faire vos réservations de machines. {SERVERNAME} est le nom du serveur que vous avez réservé, donc soit studsrv2
.info.ucl.ac.be, studsrv3.info.ucl.ac.be ou studsrv4.info.ucl.ac.be. {PATH_TO_SSH_KEY} est le chemin vers la clé ssh que vous avez reçu dans le mail contenant les informations de connexion aux serveurs d'expériences. 

Par exemple, les utilisateurs du compte guest009 qui ont réservé le serveur studsrv3.info.ucl.ac.be se connecteront comme ceci:

	sh connect_to_experiments_server.sh guest009@studsrv3.info.ucl.ac.be $HOME/.ssh/id_rsa.guest009

Un mot de passe vous sera demandé deux fois (une fois pour la passerelle et une fois pour le serveur d'expériences). Le mot de passe est a chaque fois le mot de passe de votre compte guest.

Une fois connecté sur le serveur d'expériences, vous pourrez cloner votre projet avec git, le compiler et évaluer ses performances. 



Seconde méthode: se connecter en utilisant un fichier de configuration SSH
==========================================================================

Si vous ne voulez pas utiliser le script fourni pour vous connecter au serveur, nous vous fournissons aussi un fichier de configuration ssh sur Moodle (au nom de ssh_config) à placer au chemin suivant: $HOME/.ssh/config (attention, il est important que le fichier s'appelle config et qu'il soit placé dans le dossier $HOME/.ssh !), qui demande à SSH de faire la passerelle automatiquement. Dans ce fichier, il vous faudra remplacer deux lignes indiquées par des crochets [], pour y renseigner 1) le nom d'utilisateur de votre groupe pour vous connecter aux serveurs d'expériences 2) le chemin vers votre clé ssh. Assurez-vous que les permissions de fichier de votre clé ssh sont bien mises à 600. Si ce n'est pas le cas, remédiez au problème en utilisant la commande chmod. Une fois ce fichier de configuration correctement édité et copié au bon endroit, vous pourrez vous connecter au serveur avec la commande suivante :

	ssh guest210@studsrv2.info.ucl.ac.be

L'intérêt de ce fichier de configuration est que des commandes telles que scp fonctionneront aussi en utilisant le fichier de config, pratique !



Infos pratiques
===============

Il n'y a pas de problème à réserver des slots plusieurs fois pour refaire vos expériences. Nous tenons cependant à ce que chaque groupe réserve des slots d'expériences de manière responsable vis-à-vis des autres groupes.

Nous vous invitons à évaluer les performances de votre programme le plus tôt possible pour laisser des l'espace aux groupes qui termineraient leur programme plus tard.
